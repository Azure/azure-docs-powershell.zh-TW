## 說明
**新的-AzManagementGroupDeployment** Cmdlet 會在管理群組新增部署。

若要在管理群組新增部署，請指定管理群組、位置及範本。
位置會告知 Azure 資源管理員要儲存部署資料的位置。 範本是一個 JSON 字串，其中包含要部署的個別資源。
範本包含必要資源的參數預留位置，以及可設定的屬性值，例如名稱和大小。

若要在部署中使用自訂範本，請指定 *TemplateFile* 參數或 *TemplateUri* 參數。
每個範本都有可設定屬性的參數。
若要指定範本參數的值，請指定 *TemplateParameterFile* 參數或 *TemplateParameterObject* 參數。
或者，您也可以使用在指定範本時動態新增到命令中的範本參數。
若要使用動態參數，請在命令提示字元輸入，或輸入減號 ( ) 來指示參數，然後使用 Tab 鍵來迴圈顯示可用的參數。
您在命令提示字元中輸入的範本參數值會優先于範本參數物件或檔案中的值。

若要將資源新增至資源群組，請使用 **New-AzResourceGroupDeployment** ，這會在資源群組建立部署。
若要將資源新增至訂閱，請使用 **新的-AzSubscriptionDeployment** ，這會在訂閱範圍中建立部署，以部署訂閱層級資源。
若要在租使用者範圍中新增資源，請使用 **新的-AzTenantDeployment** ，這會在租使用者範圍中建立部署。

## 示例

### 範例1：使用自訂範本和參數檔案建立部署
```
PS C:\> New-AzManagementGroupDeployment -ManagementGroupId "myMG" -Location "West US" -TemplateFile "D:\Azure\Templates\OrgSetup.json" -TemplateParameterFile "D:\Azure\Templates\OrgParms.json"
```

這個命令會使用自訂範本與磁片上的範本檔案，在管理群組「myMG」建立新的部署。
此命令會使用 *TemplateFile* 參數來指定範本和 *TemplateParameterFile* 參數，以指定包含參數和參數值的檔案。

### 範例2：使用自訂範本物件與參數檔案建立部署
```
PS C:\> $TemplateFileText = [System.IO.File]::ReadAllText("D:\Azure\Templates\OrgSetup.json")
PS C:\> $TemplateObject = ConvertFrom-Json $TemplateFileText -AsHashtable
PS C:\> New-AzManagementGroupDeployment -ManagementGroupId "myMG" -Location "West US" -TemplateObject $TemplateObject -TemplateParameterFile "D:\Azure\Templates\OrgParams.json"
```

這個命令會使用自訂範本與磁片上已轉換成記憶體中的 hashtable 的範本檔案，在管理群組「myMG」建立新的部署。
前兩個命令會讀取磁片上範本檔案的文字，並將它轉換成記憶體中的 hashtable。
最後一個命令使用 *TemplateObject* 參數來指定此 Hashtable 和 *TemplateParameterFile* 參數，以指定包含參數和參數值的檔案。
