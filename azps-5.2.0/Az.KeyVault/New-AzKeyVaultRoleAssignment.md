# New-AzKeyVaultRoleAssignment

## 摘要
在指定的範圍內，將指定的 RBAC 角色指派給指定的主體。

## 句法

### DefinitionNameSignInName (預設) 
```
New-AzKeyVaultRoleAssignment [-HsmName] <String> [-Scope <String>] -RoleDefinitionName <String>
 -SignInName <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

```

```


## 說明
使用 `New-AzKeyVaultRoleAssignment` 命令來授與存取權。
您可以在適當的範圍內指派適當的 RBAC 角色來取得存取權。
必須指定作業的主語。
若要指定使用者，請使用 SignInName 或 Azure AD ObjectId 參數。
若要指定安全性群組，請使用 Azure AD ObjectId 參數。
若要指定 Azure AD 應用程式，請使用 ApplicationId 或 ObjectId 參數。
所指派的角色，必須使用 RoleDefinitionName pr RoleDefinitionId 參數加以指定。 您可以指定授權存取的範圍。 預設為選取的訂閱。

## 示例

### 範例1
```powershell
PS C:\> New-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com

RoleDefinitionName               DisplayName                    ObjectType Scope
------------------               -----------                    ---------- -----
Managed HSM Policy Administrator User 1 (user1@microsoft.com)   User       /
```

這個範例會將角色「受管理的 HSM 策略系統管理員」指派給 user1@microsoft.com top 作用中的使用者 ""。
