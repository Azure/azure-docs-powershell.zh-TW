# Get-AzRoleAssignment

## 簡介
列出指定範圍中的 Azure RBAC 角色指派。
根據預設，它會列出所選 Azure 訂閱中所有角色指派。
使用各自的參數列出指派給特定使用者，或列出特定資源群組或資源的指派。

## 描述
使用 Get-AzRoleAssignment 命令列出範圍中有效的所有角色指派。
若沒有任何參數，此命令會返回訂閱下進行的所有角色指派。
此清單可以使用主體、角色和範圍的篩選參數進行篩選。
必須指定作業的主體。
若要指定使用者，請使用 SignInName 或 Azure AD ObjectId 參數。
若要指定安全性群組，請使用 Azure AD ObjectId 參數。
若要指定 Azure AD 應用程式，請使用 ServicePrincipalName 或 ObjectId 參數。
指派的角色必須使用 RoleDefinitionName 參數指定。
您可以指定授予存取權的範圍。
它會預設為選取的訂閱。 工作分派的範圍可以使用下列其中一個參數組合 a 來指定。
範圍 - 這是從 /subscriptions/開始的所有完整範圍 \<subscriptionId\> 。
這會篩選有效于該特定範圍的工作分派，即該範圍及以上範圍的所有工作分派。
B。
ResourceGroupName - 訂閱下任何資源組的名稱。
這會篩選指定資源群組 c 的有效工作分派。
ResourceName、ResourceType、ResourceGroupName 和 (選) ParentResource - 識別訂閱下的特定資源，並篩選該資源範圍中有效的工作分派。
若要判斷特定使用者在訂閱中擁有哪些存取權，請使用 ExpandPrincipalGroups 切換。
這會列出指派給使用者的所有角色，以及使用者所組成群組的所有角色。
使用 IncludeClassicAdministrators 切換來同時顯示訂閱系統管理員和共同管理員。

## 例子

### 範例 1
```
PS C:\> Get-AzRoleAssignment
```

列出訂閱中所有角色指派

### 範例 2
```
PS C:\> Get-AzRoleAssignment -ResourceGroupName testRG -SignInName john.doe@contoso.com
```

取得在 testRG 範圍或以上範圍內指派給使用者的所有角色，以及其為成員 john.doe@contoso.com 群組。

### 範例 3
```
PS C:\> Get-AzRoleAssignment -ServicePrincipalName "http://testapp1.com"
```

獲得指定服務主體的所有角色指派

### 範例 4
```
PS C:\> Get-AzRoleAssignment -Scope "/subscriptions/96231a05-34ce-4eb4-aa6a-70759cbb5e83/resourcegroups/rg1/providers/Microsoft.Web/sites/site1"
```

在網站 1 網站範圍中獲得角色指派。
