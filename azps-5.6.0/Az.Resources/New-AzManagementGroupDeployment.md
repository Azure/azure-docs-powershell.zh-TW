## 描述
**New-AzManagementGroupDeployment** Cmdlet 會新增管理群組的部署。

若要在管理群組新增部署，請指定管理群組、位置和範本。
位置會告知 Azure Resource Manager 要儲存部署資料的位置。 範本是 JSON 字串，其中包含要部署的個別資源。
範本包含所需資源的參數預留位置，以及可設定屬性值 ，例如名稱和大小。

若要使用自訂範本進行部署，請指定 *TemplateFile* 參數或 *TemplateUri* 參數。
每個範本都有可設定屬性的參數。
若要指定範本參數的值，請指定 *TemplateParameterFile* 參數或 *TemplateParameterObject* 參數。
或者，您也可以使用指定範本時動態新加入命令的範本參數。
若要使用動態參數，請于命令提示文字輸入參數，或輸入減號 (-) 來表示參數，並使用 Tab 鍵迴圈流覽可用的參數。
在命令提示提示輸入的範本參數值優先于範本參數物件或檔案中的值。

若要新增資源至資源群組，請使用 **New-AzResourceGroupDeployment，** 在資源群組中建立部署。
若要新增資源至訂閱，請使用 **New-AzSubscriptionDeployment，** 在訂閱範圍中建立部署，以部署訂閱層級資源。
若要在租使用者範圍中新增資源，請使用 **New-AzTenantDeployment，** 在租使用者範圍中建立部署。

## 例子

### 範例 1：使用自訂範本和參數檔案建立部署
```
PS C:\> New-AzManagementGroupDeployment -ManagementGroupId "myMG" -Location "West US" -TemplateFile "D:\Azure\Templates\OrgSetup.json" -TemplateParameterFile "D:\Azure\Templates\OrgParms.json" -Tag @{"key1"="value1"; "key2"="value2";}
```

此命令會使用自訂範本和磁片上的範本檔案，以及已定義的標籤參數，在管理群組 「myMG」建立新部署。
該命令使用 *TemplateFile* 參數來指定範本，而 *TemplateParameterFile* 參數則指定包含參數和參數值的檔案。

### 範例 2：使用 uri 和 SAS 權杖部署儲存在非公用儲存帳戶中的範本
```
PS C:\> New-AzManagementGroupDeployment -ManagementGroupId "myMG" -Location "West US" -TemplateUri "https://example.com/example.json" -QueryString "foo"
```

此命令會使用 TemplateUri 中的範本建立新部署，範本不是公用的，而且需要權杖參數來存取，這會使用 QueryString 參數提供。
執行此命令會使用 URL 有效地存取範本 https://example.com/example.json?foo 。
如果您想要在儲存帳戶中使用範本，請使用此功能，方法為提供 SAS 權杖做為 QueryString

### 範例 3：使用自訂範本物件和參數檔案建立部署
```
PS C:\> $TemplateFileText = [System.IO.File]::ReadAllText("D:\Azure\Templates\OrgSetup.json")
PS C:\> $TemplateObject = ConvertFrom-Json $TemplateFileText -AsHashtable
PS C:\> New-AzManagementGroupDeployment -ManagementGroupId "myMG" -Location "West US" -TemplateObject $TemplateObject -TemplateParameterFile "D:\Azure\Templates\OrgParams.json"
```

此命令會使用自訂範本和磁片上已轉換成記憶體雜湊表的範本檔案，在管理群組 「myMG」建立新部署。
前兩個命令會讀取磁片上範本檔案的文字，並將它轉換成記憶體內雜湊表。
最後一個命令使用 *TemplateObject* 參數來指定此雜湊表，而 *TemplateParameterFile* 參數則指定包含參數和參數值的檔案。
