# Restore-AzKeyVaultManagedStorageAccount

## 簡介
從備份檔案還原金鑰保存庫中的受管理儲存空間帳戶。

## 語法

### ByVaultName (預設) 
```
Restore-AzKeyVaultManagedStorageAccount [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultManagedStorageAccount [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Restore-AzKeyVaultManagedStorageAccount [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
**Restore-AzKeyVaultManagedStorageAccount** Cmdlet 會從備份檔案，于指定的金鑰保存庫中建立受管理的儲存帳戶。
此受管理的儲存帳戶是輸入檔案中備份受管理儲存帳戶的複本，其名稱與原始帳戶相同。
如果金鑰保存庫已經包含相同名稱的受管理儲存空間帳戶，此 Cmdlet 會失敗，而不會覆寫原始帳戶。
還原受管理儲存帳戶的金鑰保存庫，可能會與備份受管理儲存帳戶的金鑰保存庫不同。
不過，金鑰庫必須使用相同的訂閱，而且位於位於相同地理位置的 Azure (例如北美地區) 。
請參閱 Microsoft Azure 信任中心 (https://azure.microsoft.com/support/trust-center/) Azure 區域與地理位置的相互比對。

## 例子

### 範例 1：還原備份受管理的儲存空間帳戶
```powershell
PS C:\> Restore-AzKeyVaultManagedStorageAccount -VaultName 'MyKeyVault' -InputFile "C:\Backup.blob"
```
