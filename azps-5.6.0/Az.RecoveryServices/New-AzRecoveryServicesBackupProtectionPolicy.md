# New-AzRecoveryServicesBackupProtectionPolicy

## 簡介
建立備份保護原則。

## 語法

```
New-AzRecoveryServicesBackupProtectionPolicy [-Name] <String> [-WorkloadType] <WorkloadType>
 [[-BackupManagementType] <BackupManagementType>] [[-RetentionPolicy] <RetentionPolicyBase>]
 [[-SchedulePolicy] <SchedulePolicyBase>] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
**New-AzRecoveryServicesBackupProtectionPolicy** Cmdlet 在儲存庫中建立備份保護原則。
保護原則至少與一個保留原則相關聯。
保留原則會定義復原點與 Azure 備份的保留時間。
您可以使用 Cmdlet Get-AzRecoveryServicesBackupRetentionPolicyObject取得預設的保留原則。
您也可以使用 Get-AzRecoveryServicesBackupSchedulePolicyObject Cmdlet 取得預設的排程策略。
**SchedulePolicy 和** **RetentionPolicy** 物件會做為 **New-AzRecoveryServicesBackupProtectionPolicy** Cmdlet 的輸入。
使用目前的 Cmdlet 之前，使用 Set-AzRecoveryServicesVaultContext Cmdlet 來設定庫內容。

## 例子

### 範例 1：建立備份保護原則
```powershell
PS C:\> $SchPol = Get-AzRecoveryServicesBackupSchedulePolicyObject -WorkloadType "AzureVM" 
PS C:\> $SchPol.ScheduleRunTimes.Clear()
PS C:\> $Dt = Get-Date
PS C:\> $SchPol.ScheduleRunTimes.Add($Dt.ToUniversalTime())
PS C:\> $RetPol = Get-AzRecoveryServicesBackupRetentionPolicyObject -WorkloadType "AzureVM" 
PS C:\> $RetPol.DailySchedule.DurationCountInDays = 365
PS C:\> New-AzRecoveryServicesBackupProtectionPolicy -Name "NewPolicy" -WorkloadType AzureVM -RetentionPolicy $RetPol -SchedulePolicy $SchPol
```

第一個命令會獲得一個 **基準 SchedulePolicyObject，** 然後將它儲存在$SchPol變數中。
第二個命令會從任務中的排程策略移除所有$SchPol。
第三個命令使用 Get-Date Cmdlet 取得目前的日期和時間。
第四個命令會將目前的日期與時間$Dt排程策略的執行時間。
第五個命令會獲得一個基礎 **的 RetentionPolicy** 物件，然後將它儲存在$RetPol變數中。
第六個命令將保留持續時間規則設定為 365 天。
最後一個命令會根據先前命令所建立排程和保留原則，建立 **BackupProtectionPolicy** 物件。

### 範例 2

建立備份保護原則。  (自動) 

```powershell <!-- Aladdin Generated Example --> 
New-AzRecoveryServicesBackupProtectionPolicy -Name 'NewPolicy' -RetentionPolicy $RetPol -SchedulePolicy $SchPol -VaultId $vault.ID -WorkloadType AzureVM
```
