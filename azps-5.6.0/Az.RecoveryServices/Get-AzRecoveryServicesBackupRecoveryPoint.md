# Get-AzRecoveryServicesBackupRecoveryPoint

## 簡介

獲得備份專案的復原點。

## 語法

### NoFilterParameterSet (預設) 
```
Get-AzRecoveryServicesBackupRecoveryPoint [-Item] <ItemBase> [-UseSecondaryRegion] [-VaultId <String>]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### DateTimeFilter
```
Get-AzRecoveryServicesBackupRecoveryPoint [[-StartDate] <DateTime>] [[-EndDate] <DateTime>] [-Item] <ItemBase>
 [-UseSecondaryRegion] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

### RecoveryPointId
```
Get-AzRecoveryServicesBackupRecoveryPoint [-Item] <ItemBase> [-RecoveryPointId] <String>
 [[-KeyFileDownloadLocation] <String>] [-UseSecondaryRegion] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## 描述

**Get-AzRecoveryServicesBackupRecoveryPoint** Cmdlet 取得備份的 Azure 備份專案的修復點。
備份專案之後 **，AzureRmRecoveryServicesBackupRecoveryPoint** 物件會擁有一或多個修復點。
使用 -VaultId 參數設定庫上下文。

## 例子

### 範例 1：從上周取得專案的復原點數

```powershell
PS C:\> $vault = Get-AzRecoveryServicesVault -ResourceGroupName "resourceGroup" -Name "vaultName"
PS C:\> $StartDate = (Get-Date).AddDays(-7)
PS C:\> $EndDate = Get-Date
PS C:\> $Container = Get-AzRecoveryServicesBackupContainer -ContainerType AzureVM -Status Registered -Name "V2VM" -VaultId $vault.ID
PS C:\> $BackupItem = Get-AzRecoveryServicesBackupItem -ContainerType AzureVM -WorkloadType AzureVM -VaultId $vault.ID
PS C:\> $RP = Get-AzRecoveryServicesBackupRecoveryPoint -Item $BackupItem -StartDate $Startdate.ToUniversalTime() -EndDate $Enddate.ToUniversalTime() -VaultId $vault.ID
```

第一個命令會以 vaultName 為基礎，獲得庫物件。 第二個命令會獲得七天前的日期，然後將它儲存在$StartDate變數。
第三個命令會獲得今天的日期，然後將它儲存在$EndDate變數。
第四個命令會獲得 AzureVM 備份容器，並儲存在$Container變數中。 第五個命令會依據工作負載類型、保存庫Id 來獲得備份專案，然後將它儲存在$BackupItem變數。
最後一個命令會針對 $BackupItem 中的專案獲得復原點陣列，然後將它們儲存在 $RP 變數中。
