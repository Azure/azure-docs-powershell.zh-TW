## 描述
為 Azure 事件格線主題 、支援的 Azure 資源、Azure 訂閱或資源群組建立新事件訂閱。
若要建立目前所選 Azure 訂閱的事件訂閱，請指定事件訂閱名稱和目的地端點。
若要建立資源群組的事件訂閱，除了事件訂閱名稱和目的地端點之外，請指定資源組名。
若要建立 Azure 事件格線主題的事件訂閱，請同時指定主題名稱。
若要建立受支援 Azure 資源的事件訂閱，請指定資源的完整資源識別碼。 若要查看支援類型的清單，請執行 Get-AzEventGridTopicType Cmdlet。

## 例子

### 範例 1
```powershell
PS C:\> New-AzEventGridSubscription -ResourceGroup MyResourceGroup -TopicName Topic1 -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

在資源群組 MyResourceGroupName 中建立新事件訂閱 \` EventSubscription1 至 \` Azure 事件格線主題 \` Topic1，並包含 \` \` \` webaz 目的地端點 `https://requestb.in/19qlscd1` 。 此事件訂閱使用預設篩選。

### 範例 2
```powershell
PS C:\> New-AzEventGridSubscription -ResourceGroup MyResourceGroupName -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

將事件訂閱 \` EventSubscription1 新建立至 \` 具有 web一文目的地端點的資源群組 \` MyResourceGroupName。 \` `https://requestb.in/19qlscd1` 此事件訂閱使用預設篩選。

### 範例 3
```powershell
PS C:\> New-AzEventGridSubscription -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

將事件訂閱 EventSubscription1 新建立至目前選取的 Azure 訂閱，並包含 \` \` webaz 目的地端點 `https://requestb.in/19qlscd1` 。 此事件訂閱使用預設篩選。

### 範例 4
```powershell
PS C:\> $includedEventTypes = "Microsoft.Resources.ResourceWriteFailure", "Microsoft.Resources.ResourceWriteSuccess"
PS C:\> $labels = "Finance", "HR"
PS C:\> New-AzEventGridSubscription -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1 -SubjectBeginsWith "TestPrefix" -SubjectEndsWith "TestSuffix" -IncludedEventType $includedEventTypes -Label $labels
```

將事件訂閱 EventSubscription1 新建立至目前選取的 Azure 訂閱，並包含 \` \` webaz 目的地端點 `https://requestb.in/19qlscd1` 。 此事件訂閱會指定事件種類和主題的額外篩選，且只有符合這些篩選的事件會傳送至目的地端點。

### 範例 5
```powershell
PS C:\> New-AzEventGridSubscription -EventSubscriptionName EventSubscription1 -EndpointType "eventhub" -Endpoint "/subscriptions/55f3dcd4-cac7-43b4-990b-a139d62a1eb2/resourceGroups/TestRG/providers/Microsoft.EventHub/namespaces/ContosoNamespace/eventhubs/EH1"
```

將新事件訂閱 EventSubscription1 建立至目前選取的 Azure 訂閱，並指定事件 \` \` 中樞做為事件的目的地。 此事件訂閱使用預設篩選。

### 範例 6
```powershell
PS C:\> New-AzEventGridSubscription -ResourceId "/subscriptions/55f3dcd4-cac7-43b4-990b-a139d62a1eb2/resourceGroups/TestRG/providers/Microsoft.EventHub/namespaces/ContosoNamespace" -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

建立事件訂閱 \` EventSubscription1 至 \` 具有指定 web一文目的地端點的 EventHub 命名空間 `https://requestb.in/19qlscd1` 。 此事件訂閱使用預設篩選。

### -AdvancedFilter
指定用於屬性型篩選之多個雜湊表值的陣列的進一級篩選。 每個雜湊表值都有下列索引鍵值資訊：運算、索引鍵和值或值。 運算子可以是下列其中一個值：NumberIn、NumberNotIn、NumberLessQual、NumberGreaterQual、NumberLessQualOrEquals、NumberGreaterQualOrEquals、BoolEquals、StringIn、StringNotIn、StringBeginsWith、StringEndsWith 或 StringContains。 Key 代表已使用進位篩選原則的負載屬性。 最後，值或值代表要相符的值或一組值。 這可以是對應類型的單一值或值陣列。 做為進一步篩選參數的範例：$AdvancedFilters=@ ($AdvFilter 1、$AdvFilter 2) 其中 $AdvFilter 1=@{operator="NumberIn"; key="Data.Key1";Values=@ (1，2) } 和 $AdvFilter 2=@{operator="StringBringsWith"; key="Subject";Values=@ ("SubjectPrefix1"，"SubjectPrefix2") }
