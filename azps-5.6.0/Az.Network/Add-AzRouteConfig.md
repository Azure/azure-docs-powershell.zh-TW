# Add-AzRouteConfig

## 簡介
新增路由至路由資料表。

## 語法

```
Add-AzRouteConfig -RouteTable <PSRouteTable> [-Name <String>] [-AddressPrefix <String>] [-NextHopType <String>]
 [-NextHopIpAddress <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## 描述
**Add-AzRouteConfig** Cmdlet 會新增路由至 Azure 路由資料表。

## 例子

### 範例 1：新增路由至路由資料表
```powershell
PS C:\>$RouteTable = Get-AzRouteTable -ResourceGroupName "ResourceGroup11" -Name "RouteTable01"
PS C:\> Add-AzRouteConfig -Name "Route13" -AddressPrefix 10.3.0.0/16 -NextHopType "VnetLocal" -RouteTable $RouteTable
```

第一個命令會使用 Get-AzRouteTable Cmdlet，獲得名為 RouteTable01 的路由資料表。
命令會儲存資料表$RouteTable變數。
第二個命令會將名為 Route13 的路由新增到儲存在 $RouteTable 的路由資料表。
此路由會轉送封包到本地虛擬網路。

### -NextHopType
指定此路由如何轉送封包。
此參數可接受的值為：
- 互聯網。
Azure 提供的預設網際網路閘道。 
- 沒有。
如果您指定此值，路由不會轉送封包。 
- VirtualAppliance。
您新加入 Azure 虛擬網路的虛擬裝置。 
- VirtualNetworkGateway。
Azure 伺服器對伺服器虛擬專用網路閘道。 
- VnetLocal。
本地虛擬網路。
如果您在同一個虛擬網路中有兩個子網，10.1.0.0/0/16 和 10.2.0.0/16，請選取每個子網的 VnetLocal 值以轉往另一個子網。
