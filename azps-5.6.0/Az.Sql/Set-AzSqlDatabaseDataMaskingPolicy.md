# Set-AzSqlDatabaseDataMaskingPolicy

## 簡介
設定資料庫的資料遮罩。

## 語法

```
Set-AzSqlDatabaseDataMaskingPolicy [-PassThru] [-PrivilegedUsers <String>] [-DataMaskingState <String>]
 [-ServerName] <String> [-DatabaseName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 描述
**Set-AzSqlDatabaseDataMaskingPolicy** Cmdlet 會設定 Azure SQL 資料庫的資料遮罩策略。
若要使用此 Cmdlet，請使用 *ResourceGroupName、ServerName* 和 *DatabaseName* 參數來識別資料庫。
您可以設定 *DataMaskingState* 參數來指定是否啟用或停用資料遮罩作業。
如果 Cmdlet 成功且 *使用 PassThru* 參數，除了資料庫識別碼之外，會傳回描述目前資料遮罩策略的物件。
資料庫識別碼包含但不限於 **ResourceGroupName、ServerName** 和 **DatabaseName。**
Azure 上的 SQL Server 延伸資料庫服務也支援此 Cmdlet。

## 例子

### 範例 1：設定資料庫的資料遮罩策略
```
PS C:\>Set-AzSqlDatabaseDataMaskingPolicy -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01 -PrivilegedUsers "public" -DataMaskingState "Enabled"
```

此命令會針對伺服器名為 database01 的資料庫設定資料遮罩策略。
