# New-AzLogicApp

## 簡介
在資源群組中建立邏輯應用程式。

## 語法

### LogicAppWithDefinitionParameterSet
```
New-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 -Definition <Object> [-IntegrationAccountId <String>] [-Parameters <Object>] [-ParameterFilePath <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### LogicAppWithDefinitionFileParameterSet
```
New-AzLogicApp -ResourceGroupName <String> -Name <String> -Location <String> [-State <String>]
 -DefinitionFilePath <String> [-IntegrationAccountId <String>] [-Parameters <Object>]
 [-ParameterFilePath <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## 描述
**New-AzLogicApp** Cmdlet 會使用邏輯應用程式功能建立邏輯應用程式。
邏輯應用程式是邏輯 App 定義中定義的動作或觸發程式集合。
此 Cmdlet 會返回 **工作流程** 物件。
您可以指定名稱、位置、邏輯應用程式定義、資源群組和計畫，以建立邏輯應用程式。
邏輯應用程式定義和參數的格式設定為 JavaScript 物件標記法 (JSON) 。
您可以使用邏輯應用程式做為定義和參數的範本。
此模組支援動態參數。
若要使用動態參數，請于命令中輸入。
若要探索動態參數的名稱，在 Cmdlet 名稱之後輸入連字號 (-) ，然後重複按 Tab 鍵以迴圈流覽可用的參數。
如果您省略必要的範本參數，Cmdlet 會提示您輸入值。
在命令列中指定的範本參數檔案值優先于範本參數物件的範本參數值。
