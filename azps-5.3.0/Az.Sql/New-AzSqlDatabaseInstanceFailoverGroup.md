# New-AzSqlDatabaseInstanceFailoverGroup

## 摘要
這個命令會建立新的 Azure SQL 資料庫實例 [容錯移轉] 群組。

## 句法

```
New-AzSqlDatabaseInstanceFailoverGroup [-Name] <String> [-PartnerResourceGroupName <String>]
 -PartnerRegion <String> -PrimaryManagedInstanceName <String> -PartnerManagedInstanceName <String>
 [-PartnerSubscriptionId <String>] [-FailoverPolicy <String>] [-GracePeriodWithDataLossHours <Int32>]
 [-AllowReadOnlyFailoverToPrimary <String>] [-ResourceGroupName] <String> [-Location] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## 說明
在指定區域（具有已記下的受管理實例對）之間建立新的 Azure SQL 資料庫實例容錯移轉群組。

在 Name 中建立兩個 Azure SQL Database TDS 端點 SqlDatabaseDnsSuffix (例如，Name.database.windows.net) 和 Name. SqlDatabaseDnsSuffix。 這些端點可用來連接至容錯移轉群組的主要和次要區域。 如果主要區域受中斷所影響，端點和資料庫的自動容錯移轉會根據實例容錯移轉群組的容錯移轉原則和寬限期來觸發。

在實例容錯移轉群組功能的預覽期間，"-GracePeriodWithDataLossHours" 參數只支援大於或等於1小時的值。
