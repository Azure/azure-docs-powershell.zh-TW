## 說明
修改 Templace 規格。如果指定名稱與/或特定版本的範本規格尚不存在，則會建立。

在修改範本規格版本的 ARM 範本內容時，內容可能是來自原始的 JSON 字串， (使用 **UpdateVersionByNameFromJsonParameterSet** 參數集) 或從指定的 json 檔案（ (使用 **UpdateVersionByNameFromJsonFileParameterSet** 參數集) ）。

## 示例

### 範例1：
```powershell
PS C:\> $templateJson = @"
{
    "$schema": "http://schema.management.azure.com/schemas/2015-01-01/deploymentTemplate.json#",
    "contentVersion": "1.0.0.0",
    "parameters": {},
    "resources": []
}
"@
PS C:\> Set-AzTemplateSpec -ResourceGroupName 'myRG' -Name 'myTemplateSpec' -Version 'v1.0' -Location 'West US' -TemplateJson $templateJson
```

修改名為 "myTemplateSpec" 的範本規格 "v 1.0" 版本。 指定的版本將 $templateJson 為版本的 ARM 範本內容。 如果根範本規格與/或版本還不存在，就會建立它們。


**筆記** 

* 這個範例中的 ARM 範本是不含實際資源的 [非 op]。
* 只有範本規格不存在時，才需要位置

### 範例2：
```powershell
PS C:\> Set-AzTemplateSpec -ResourceGroupName 'myRG' -Name 'myTemplateSpec' -Version 'v2.0' -Location 'West US' -TemplateFile 'myTemplateContent.json'
```

修改名為 "myTemplateSpec" 的範本規格 "v 2.0" 版本。 指定的版本會將本機檔案 "myTemplateContent.js的內容] 做為版本的 ARM 範本內容。 如果根範本規格與/或版本還不存在，就會建立它們。

**注意：** 只有範本規格不存在時，才需要位置

### 範例3：
```powershell
PS C:\> Set-AzTemplateSpec -ResourceGroupName 'myRG' -Name 'myTemplateSpec'  -Location 'West US' -Description 'My updated Template Spec'
```

修改資源群組 "myRG" 中名為 "myTemplateSpec" 的範本規格的描述。 如果範本規格還不存在，則會建立。

**注意：** 只有範本規格不存在時，才需要位置
